### Changed

- **The write path enforces the CAS co-write invariant at runtime** (MMR-303).
  Every document a write plan mutates must carry at least one
  `expected_old_value`-guarded op; the writer now refuses an unguarded plan
  before it reaches `vault.apply` instead of trusting each verb's discipline
  by review alone. To keep the first tag on an untagged entity guarded,
  tagging now co-writes the `updated_at` stamp on node and project targets —
  tagging counts as activity, so re-tagging also refreshes `updated_at`.
