### Added

- **`mimir doctor` detects cross-directory duplicate artifact stems** (MMR-282). A
  new diagnostic reports when one artifact id (`KEY-aN`) resolves to more than one
  document — a hand-misplaced or externally-moved `KEY-aN.md` outside
  `KEY/artifacts/`, which since per-directory `{{seq}}` allocation
  ([ADR 0016](docs/decisions/0016-norn-vault-system-of-record.md) MMR-196 refinement)
  frees its number for a later create to mint the same stem canonically. Unlike
  work-state identities, artifacts read verbatim, so nothing caught this at the
  reader; the check names every physical path in its evidence and surfaces in the
  console record-health facet (a duplicate hides one document on the canonical
  point-read, the same class as identity uniqueness). Detection only — which of the
  two documents survives is a human call, so `--fix` skips it
  ([ADR 0023](docs/decisions/0023-concurrency-and-safety-posture.md): doctor
  remediates, it does not guard the allocator).
- **The interior sequence-gap check now covers artifact sequences** (MMR-282,
  extending MMR-197). Artifacts allocate identically to nodes and seeds since
  MMR-196, so the seq-gap check reports a missing artifact number below a project's
  max the same way — informational, non-gating, and not repairable. A number is
  occupied by any document bearing that stem anywhere (canonical or misplaced), so a
  misplaced artifact never reads as a phantom gap; the duplicate-stem check owns the
  misplacement story.
