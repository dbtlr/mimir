### Added

- **Doctor issue-count trailer on `next` / `list`** (MMR-184). When the
  tolerant reader drops or notes a record while building the working set a
  selection reads over — a duplicate stem, a dangling or cycle-closing edge, a
  missing project, an invalid lifecycle — `next` and `list` now print
  `⚠ N issues — run mimir doctor` as a stderr note, nudging an agent's boot
  orientation toward vault issues. The count is the load's own byproduct (the
  shared validator's drop tally over the working-set graph), never a fresh
  `mimir doctor` pass — so doctor, which also runs seed and per-document
  checks, may report more. Stdout stays untouched: machine formats
  (`ids` / `json` / `jsonl`) carry no trailer, and a clean vault prints
  nothing.
