### Added

- **Doctor issue-count trailer on `next` / `list`** (MMR-184). When the
  tolerant reader drops or notes a record while building the working set a
  selection reads over — a parse failure, a duplicate stem, a dangling edge —
  `next` and `list` now print `⚠ N issues — run mimir doctor` as a stderr note,
  nudging an agent's boot orientation toward vault issues without a fresh
  `mimir doctor` pass. The count is the load's own byproduct (the same
  `validate()` tally doctor's `ctx.dropped` draws on); stdout stays untouched,
  so machine formats (`ids` / `json` / `jsonl`) carry no trailer, and a clean
  vault prints nothing.
